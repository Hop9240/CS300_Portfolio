# CS300_Portfolio
What was the problem you were solving in the projects for this course?
The problem I was solving was how to structure course information in a way that would benefit academic advisors. The program had to read course data from a file, store that data in an appropriate data structure, print the course list in alphanumeric order, and allow the user to search for specific courses as efficiently as possible.
How did you approach the problem? Consider why data structures are important to understand.
I approached the problem by comparing different data structures and evaluating which one would be the most efficient for searching and organizing course information. I focused on how quickly each structure could insert and search for specific courses. Understanding data structures is important because it affects overall performance.
How did you overcome any roadblocks you encountered while going through the activities or project?
I overcame roadblocks by going back to previous assignments and reviewing the patterns that were used in those assignments. When I compared them to the new assignment, I could see the similarities, which made it easier to understand the problem and get past the roadblocks.
How has your work on this project expanded your approach to designing software and developing programs?
My approach expanded because this project exposed me to different interpretations of programming languages. I was able to see how the structure of C++ is both similar to and different from Java. I now have more knowledge on how to read pseudocode and implement it in either Java or C++, which was a big concern for me before taking this class.
How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?
The main thing I learned in this class was structure — not just the data structures we used to store bids and courses, but the overall structure of organizing an entire program. We did repetitive work throughout the course, and many of the ideas stayed the same with only small changes depending on the data structure being used. The order of the steps we followed did not change much. This helped me understand how to write readable and maintainable code, because repetition builds familiarity and confidence. The more I practiced structuring programs properly, the more comfortable I became with writing organized and adaptable code.
